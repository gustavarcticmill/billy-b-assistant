Create `webconfig/static/js/wake-word-panel.js` as a complete IIFE module (per WWUI-10). Follow the AudioPanel pattern: `const WakeWordPanel = (() => { ... return { init }; })();`

The module must implement ALL of the following behaviors. Use camelCase for all variables and functions per project convention.

**1. Private state variables:**
```javascript
let statusInterval = null;    // 3s status poll timer
let eventInterval = null;     // 2s event poll timer
let stopArmed = false;        // click-to-arm state for stop button
let stopTimer = null;         // 2s auto-revert timer for stop arm
let calState = { ambient: null, phrase: null, suggested: null }; // calibration data
```

**2. init() function (public, called from init.js):**
- Bind all button click handlers (simulate, stop, refresh, cal ambient, cal phrase, cal apply)
- Bind enable toggle change handler
- Start both polling intervals: `setInterval(pollStatus, 3000)` and `setInterval(pollEvents, 2000)` (per D-01, D-06)
- Run initial `pollStatus()` immediately to set badge on load
- Polling runs continuously, never paused on collapse (per UI-SPEC rationale: badge stays current, events drain server buffer)

**3. pollStatus() function (per D-06, WWUI-01):**
- `GET /wake-word/status`
- On success: call `updateStatusBadge(data)` and sync enable toggle checkbox (`ww-enable-toggle.checked = data.enabled`)
- On error: set badge to "Unavailable" state
- Badge state derivation logic (in order of precedence):
  - `data.error` is truthy -> state = "error"
  - `data.enabled === false` -> state = "disabled"
  - `data.running === false && data.enabled === true` -> state = "unavailable"
  - `data.running === true` -> state = "listening"
- Badge color map (from UI-SPEC):
  - listening: bg=`bg-emerald-500/20`, text=`text-emerald-400`, dot=`bg-emerald-400`, pulse=true
  - disabled: bg=`bg-zinc-600/20`, text=`text-zinc-400`, dot=`bg-zinc-400`, pulse=false
  - paused: bg=`bg-amber-500/20`, text=`text-amber-400`, dot=`bg-amber-400`, pulse=false
  - error: bg=`bg-rose-500/20`, text=`text-rose-400`, dot=`bg-rose-400`, pulse=false
  - unavailable: bg=`bg-zinc-600/20`, text=`text-zinc-500`, dot=`bg-zinc-500`, pulse=false
- Set badge className directly (clear all classes, set fresh): `badge.className = \`flex items-center gap-2 px-3 py-1.5 rounded-full \${style.bg}\``
- Set dot className: `dot.className = \`w-2 h-2 rounded-full \${style.dot}\${style.pulse ? " animate-pulse" : ""}\``
- Set text className and textContent (capitalize first letter of state)

**4. Enable/Disable toggle handler (WWUI-01):**
- On `change` event of `ww-enable-toggle`:
- Read new `checked` state
- `POST /wake-word/runtime-config` with `{ "enabled": checked }`
- On success: `showNotification(checked ? "Wake word detection enabled" : "Wake word detection disabled", checked ? "success" : "info")`
- On error: REVERT checkbox to previous state (`ww-enable-toggle.checked = !checked`), `showNotification("Failed to update wake word: " + err.message, "error")`

**5. Action buttons (WWUI-04):**

Simulate Detection (`ww-simulate-btn`):
- `POST /wake-word/test` with `{ "action": "simulate" }`
- Success: `showNotification("Detection simulated", "success")`
- Error: `showNotification("Simulation failed: " + err.message, "error")`

Stop Session (`ww-stop-btn`) with click-to-arm (per UI-SPEC):
- First click: set `stopArmed = true`, change button HTML to `<span class="material-icons text-base">stop</span> Hold to Stop`, change classes to `bg-rose-500/20 text-rose-400` (remove `bg-zinc-800 hover:bg-zinc-700 text-white`), set 2s revert timer
- Second click within 2s window: clear timer, reset `stopArmed = false`, reset button styling, execute `POST /wake-word/test` with `{ "action": "stop" }`
- Timer expires: reset button to default state
- Success notification: `showNotification("Session stopped", "info")`
- Error notification: `showNotification("Stop failed: " + err.message, "error")`

Refresh Status (`ww-refresh-btn`):
- Call `pollStatus()` (which does `GET /wake-word/status`)
- On success: `showNotification("Status refreshed", "info")`
- Wrap separately so the notification only fires for manual refresh, not polling

**6. Event log polling and display (WWUI-05, per D-01, D-02):**

pollEvents():
- `GET /wake-word/events`
- If `data.events.length === 0` return early
- Hide `ww-event-empty` div (add `hidden` class)
- Check auto-scroll: `wasAtTop = log.scrollTop === 0` BEFORE prepending
- For each event, create an entry element and prepend to `ww-event-log`:
  - Entry structure: div with `flex items-start gap-2 py-2 px-3 border-b border-zinc-700/50 last:border-0`
  - Icon span: Material Icon based on event type mapping:
    - "detection" -> icon `mic`, color `text-emerald-400`
    - "error" -> icon `error_outline`, color `text-rose-400`
    - "started" or "stopped" -> icon `power_settings_new`, color `text-amber-400`
    - default -> icon `info`, color `text-slate-400`
  - Content div: type text + timestamp + detail
  - Timestamp format: `HH:MM:SS` 24-hour local time. Parse ISO timestamp with `new Date(event.timestamp)` then format with `toLocaleTimeString('en-GB', { hour: '2-digit', minute: '2-digit', second: '2-digit' })`
- Cap at 50 entries: while `log.children.length > 51` (50 entries + 1 hidden empty div), remove `log.lastChild`
- Update count: `ww-event-count.textContent = \`\${count} events\``
- Auto-scroll: if `wasAtTop`, set `log.scrollTop = 0` (keep user at top seeing newest)

**7. Calibration wizard (WWUI-06, WWUI-07, WWUI-08, per D-03, D-04, D-05):**

Constants: `CAL_ADVANCE_DELAY = 1500` (ms between steps)

Calibration ambient button (`ww-cal-ambient-btn`) handler:
1. Disable button, show recording state (id=`ww-cal-recording`, unhide; hide step1 button area)
2. Set status text: "Recording ambient noise..." (id=`ww-cal-status`)
3. Start client-side countdown: "3 seconds remaining" -> "2 seconds remaining" -> "1 second remaining" (id=`ww-cal-countdown`), 1s interval
4. `POST /wake-word/calibrate` with `{ "mode": "ambient" }`
5. On response: store `calState.ambient = data`, `calState.suggested = data.suggested_threshold`
6. Show results grid (id=`ww-cal-results`, unhide): populate `ww-cal-ambient-rms` with `data.rms_mean`, `ww-cal-peak-rms` with `data.rms_peak`, `ww-cal-suggested-threshold` with `data.suggested_threshold`
7. Hide recording state
8. After `CAL_ADVANCE_DELAY` (1500ms), auto-advance to Step 2: update step indicators (step 1 completed with check icon, step 2 active), show step2 content, hide step1 content

Calibration phrase button (`ww-cal-phrase-btn`) handler:
1. Same recording UI pattern. Status text: "Say the wake word now..."
2. `POST /wake-word/calibrate` with `{ "mode": "phrase" }`
3. On response: store `calState.phrase = data`
4. Show results: update `ww-cal-phrase-peak` with `data.rms_peak`. Show full 2x2 results grid
5. After `CAL_ADVANCE_DELAY`, auto-advance to Step 3: update step indicators (step 2 completed, step 3 active), show step3 content, hide step2 content
6. Populate `ww-cal-suggested-value` with `calState.suggested`

Step indicator update function:
- Active step: circle classes `bg-emerald-500 text-white`, label classes `text-slate-300`
- Completed step: circle classes `bg-emerald-500 text-white`, content replaced with `check` icon, label classes `text-slate-300`
- Future step: circle classes `bg-zinc-700 text-zinc-400`, label classes `text-zinc-500`

Apply Suggestions button (`ww-cal-apply-btn`) handler (per D-05):
1. Read `ww-cal-persist` checkbox state
2. If persist checked: `POST /wake-word/calibrate/apply` with `{ "threshold": calState.suggested }`
3. If persist unchecked: `POST /wake-word/runtime-config` with `{ "threshold": calState.suggested }`
4. On success: `showNotification("Calibration applied", "success")`. Reset wizard to Step 1 initial state (reset all step indicators, show step1 content, hide step2/step3/results/recording, clear calState)
5. On error: `showNotification("Failed to apply calibration: " + err.message, "error")`. Keep wizard at Step 3 for retry.

**All fetch calls** must use try/catch with `showNotification` on error, following the exact pattern from audio-panel.js:
```javascript
try {
    const res = await fetch(url, options);
    if (!res.ok) throw new Error(`HTTP ${res.status}`);
    const data = await res.json();
    if (data.error) throw new Error(data.error);
    // handle success
} catch (err) {
    showNotification("Error message: " + err.message, "error");
}
```
